# najva-messenger

یک پیام‌رسان خود-میزبان، متن‌باز و کاملاً رمزنگاری‌شده سرتاسری (ایمن) — گپ متنی، تماس صوتی و تصویری، اشتراک‌گذاری فایل‌های رمزنگاری‌شده و اعلان‌های لحظه‌ای — که کاملاً روی سروری که خودتان کنترل می‌کنید اجرا می‌شود.

پیام‌ها پیش از خروج از دستگاه در مرورگر رمزنگاری می‌شوند و سرور هرگز کلیدی برای خواندن آن‌ها ندارد. این سیستم تنها کلیدهای بسته‌بندی‌شده و متن‌های رمزنگاری‌شده را ذخیره می‌کند و نه هیچ چیز دیگر. با اجرا کردن تنها یک دستور روی سرور خام Ubuntu 24.04، تمام این مجموعه راه‌اندازی شده، چهار سوال از شما پرسیده و گواهی SSL (TLS) صادر می‌کند.

## چرا این پروژه ساخته شد؟

بیشتر پلتفرم‌های پیام‌رسان خود-میزبان شما را بر سر یک دوراهی قرار می‌دهند: یا نصب آن ساده است اما رمزنگاری فقط در لایه انتقال صورت می‌گیرد، یا رمزنگاری واقعی است اما نصب آن مستلزم روزها درگیری با فایل‌های پیچیده Docker و Compose است. `najva-messenger` برای حالتی ساخته شده که شما هر دو را می‌خواهید:

- رمزنگاری محتوای پیام‌ها و فایل‌ها در سمت کاربر،
- برقراری تماس صوتی و تصویری از طریق SFU و سرور اختصاصی TURN، و
- نصب آسان تنها با اجرای یک دستور `curl | bash` روی سرور Ubuntu 24.04،

بدون اینکه داده‌های شما به زیرساخت شخص دیگری منتقل شود.

## سرویس‌ها

پنج کانتینر در کنار هم توسط Docker Compose اجرا می‌شوند:

| سرویس | نقش |
| --- | --- |
| `client` | رابط کاربری سمت‌کاربر (React + Vite SPA)، Service Worker، صف ارسال آفلاین، پشتیبانی از دو زبان انگلیسی/فارسی با کامل‌ترین حالت راست‌چین (RTL) |
| `server` | API مبتنی بر Express، دیتابیس اصلی PostgreSQL از طریق Prisma، مدیریت حضور آنلاین و Redis، ارسال پیام‌ها با Socket.IO |
| `media-server` | سرور اختصاصی mediasoup (SFU) برای تماس‌های صوتی و تصویری دو‌نفره و گروهی |
| `turn` | سرور coturn برای برقراری ارتباط در شبکه‌های محدود یا دارای NAT |
| `nginx` | نقطه ورودی واحد؛ مدیریت TLS، پروکسی API، WebSocket و فایل‌های استاتیک |

بخش مدیریت — مدیریت کاربران، سهمیه فضا، تیکت‌های پشتیبانی، آمار سرور — بخشی از همان برنامه وب است که توسط پرچم مدیریت محافظت می‌شود. **پس از ورود با حساب مدیر، می‌توانید از مسیر `/admin` (مانند `http://<domain_or_ip>/admin`) به پنل مدیریت دسترسی داشته باشید.**

## مدل رمزنگاری

هر پیام با کلیدی رمزنگاری می‌شود که فقط شرکت‌کنندگان گفتگو آن را دارند. این کلید با کلید حساب هر کاربر بسته‌بندی می‌شود. کلید حساب نیز از یک کلید رمزنگاری اصلی (KEK) مشتق می‌شود که خود از رمز عبور کاربر به دست می‌آید. سرور تنها کلیدهای بسته‌بندی‌شده و یک تاییدکننده ورود را ذخیره می‌کند — سرور هرگز رمز عبور، KEK یا متن اصلی پیام‌ها را نمی‌بیند. فایل‌ها نیز پیش از بارگذاری رمزنگاری می‌شوند، بنابراین فایل‌های ذخیره‌شده روی دیسک کاملاً نامفهوم هستند.

قابلیت Passkey PRF به عنوان یک مسیر ثانویه برای بازیابی حساب استفاده می‌شود و اثر انگشت‌های هویت به دو نفر اجازه می‌دهند یکدیگر را بیرون از سیستم تایید کنند.

> **توجه:** چون رمز عبور کلیدهای اصلی را می‌سازد، در صورت فراموشی رمز عبور امکان بازنشانی آن از سمت سرور بدون از دست رفتن تاریخچه پیام‌ها وجود ندارد. این ویژگی امنیتی سیستم است. بنابراین گزینه بازنشانی مدیر در منوی `najva` حساب را از نو می‌سازد.

برای اطلاعات بیشتر، سند [docs/ENCRYPTION.md](docs/ENCRYPTION.md) را مطالعه کنید.

## نصب

نصب روی Ubuntu روش اصلی و پیشنهادی برای راه‌اندازی سرور است. همچنین [راهنمای نصب روی ویدوز](#نصب-روی-ویندوز) برای اجرای محلی وجود دارد.

### پیش‌نیازها

- Ubuntu 24.04 با یک IP عمومی و دسترسی root
- یک دامنه اشاره‌کننده به IP (در صورت نیاز به HTTPS)

Docker به طور خودکار در صورت عدم وجود نصب می‌شود.

### نصب با یک دستور

```bash
curl -fsSL https://raw.githubusercontent.com/MatinMHF/najva-messenger/main/install.sh | sudo bash
```

اسکریپت نصب پروژه را در `/opt/najva` کلون کرده، تمام کلیدهای امنیتی را به صورت تصادفی تولید می‌کند و ۴ سوال می‌پرسد:

| سوال | مقدار پیش‌فرض | توضیحات |
| --- | --- | --- |
| پورت HTTP | `80` | برای تایید کلید Enter را بزنید |
| پورت HTTPS | `443` | برای تایید کلید Enter را بزنید |
| نام کاربری و رمز عبور مدیر | `admin` | رمز عبور باید حداقل ۸ کاراکتر باشد |
| دامنه | *(هیچ)* | در صورت عدم وجود دامنه و استفاده از IP خالی بگذارید |
| ایمیل Let's Encrypt | `admin@<domain>` | فقط در صورت وارد کردن دامنه پرسیده می‌شود |

پس از پایان موفقیت‌آمیز نصب، آدرس‌های زیر در دسترس خواهند بود:
- **آدرس پیام‌رسان:** `http://<domain_or_ip>`
- **پنل مدیریت (Admin Panel):** `http://<domain_or_ip>/admin`

## مدیریت

اسکریپت نصب یک دستور `najva` روی سرور قرار می‌دهد. با اجرای آن منوی زیر باز می‌شود:

| گزینه | کاربرد |
| --- | --- |
| تلاش مجدد برای دریافت SSL | دریافت مجدد گواهی SSL از certbot |
| بازنشانی نام کاربری و رمز عبور مدیر | ساخت مجدد حساب مدیر |
| راه‌اندازی مجدد سرویس | `docker compose restart` |
| متوقف کردن سرویس | `docker compose down` |
| شروع سرویس | `docker compose up -d` |
| وضعیت | نمایش وضعیت کانتینرها، دامنه و پورت‌ها |
| گزارش‌ها (Logs) | مشاهده لاگ کانتینرها |
| بررسی بروزرسانی | مقایسه نسخه فعلی با نسخه اصلی main و بروزرسانی |
| حذف کامل | پاکسازی کامل کانتینرها و داده‌ها |

## بروزرسانی

نسخه نصب‌شده در فایل `VERSION` ثبت می‌شود. برای بروزرسانی کافیست دستور زیر را اجرا کنید:

```bash
najva            # انتخاب گزینه "Check for updates"
```

## نصب روی ویندوز

برای اجرای محلی Najva در ویندوز، Power Shell را به صورت Administrator باز کرده و دستور زیر را اجرا کنید:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
irm https://raw.githubusercontent.com/MatinMHF/najva-messenger/main/install/install-windows.ps1 | iex
```

برنامه روی `http://localhost` و پنل مدیریت روی `http://localhost/admin` اجرا خواهد شد.

## پیکربندی

تمام تنظیمات در فایل `.env` قرار دارند که توسط اسکریپت نصب مدیریت می‌شوند.

## توسعه محلی

```bash
git clone https://github.com/MatinMHF/najva-messenger.git
cd najva-messenger
cp .env.example .env
docker compose -f docker-compose.dev.yml up
```
